# 要求仕様書

## 1. プロジェクト背景

トレーニングジムにおいて、トレーナーが顧客のトレーニング記録を紙やスプレッドシートで管理している状況を改善する。トレーニング中にその場で記録・閲覧できるWebアプリを構築し、記録業務の効率化と記録精度の向上を図る。

## 2. 利用者

### 現在のスコープ
| ロール | 説明 |
|--------|------|
| トレーナー | ジムスタッフ。顧客のトレーニング記録の作成・閲覧・編集・削除すべてを担う |

### 将来の拡張（設計時に考慮）
| ロール | 説明 |
|--------|------|
| 顧客 | 自分のトレーニング記録を閲覧のみ可能（編集不可） |

## 3. 利用端末

### 端末別の利用シーン

| 端末 | 主な用途 | 補足 |
|------|---------|------|
| iPhone（スマホ） | トレーニング中のセッション記録・過去記録の参照 | **主要端末**。顧客と並んでその場で操作する |
| PC（デスクトップ/ノート） | 顧客情報の登録・種目マスタの整備・詳細メモの入力 | 事務作業的な操作に利用 |

### 端末ごとの要件

**iPhone（スマートフォン）**
- iOSのSafariで快適に動作すること
- 片手操作・グローブ着用時でも操作しやすいタップターゲット（最低44px）
- ホームバー・ノッチのセーフエリアを考慮したレイアウト（`env(safe-area-inset-*)` 対応）
- 入力フィールドのフォントサイズは16px以上（iOS Safariのズーム防止）
- 数値入力（重量・回数）にはテンキーキーボードを表示（`inputMode="numeric"`）

**PC**
- 広い画面幅を活かした情報密度の高いレイアウト（顧客一覧・種目一覧など）
- キーボードによる快適なテキスト入力（メモ・詳細情報の登録）
- テーブル形式での一覧表示など、PCに最適化したビュー

## 4. 主なユースケース

### UC-01: トレーニングセッションの記録
- トレーナーが顧客と一緒にアプリを見ながら種目を選ぶ
- セット数・回数・重量をその場で入力する
- フォームの注意点などをメモとして残す

### UC-02: 過去記録の参照
- 前回セッションの記録を確認しながら今日の目標重量を決める
- 顧客ごとのトレーニング履歴を時系列で閲覧する

### UC-03: 顧客管理
- トレーナーが担当顧客を登録・管理する
- 顧客情報は最小限（名前・ニックネームのみ）

### UC-04: 種目マスタ管理
- よく使う種目をマスタとして登録・管理する
- セッション時に素早く種目を選択できる

## 5. 機能要件

### 必須機能（MVP）
- [ ] トレーナー認証（メール/パスワード）
- [ ] 顧客の登録・一覧・編集・削除（CRUD）
- [ ] 種目マスタの登録・一覧・編集・削除（CRUD）
- [ ] トレーニングセッションの作成・記録・閲覧
  - セット単位での重量・回数の記録
  - トレーナーメモの記録
  - 担当トレーナー名の自動記録
- [ ] 過去セッションの履歴閲覧（顧客別・日付順）

### 将来機能（拡張）
- 顧客ログイン（自分の記録閲覧のみ）
- 写真・動画のアップロード（Supabase Storageを使う場合は要検討）
- トレーニングプランの事前作成
- 体重・体脂肪率などの体組成記録
- 統計・グラフ表示

## 6. 非機能要件

### パフォーマンス
- 顧客一覧・セッション一覧の表示: 2秒以内
- セット記録の保存（onBlur後のUPSERT完了）: 1秒以内（トレーニング中断を防ぐ）

### ユーザビリティ
- **iPhone（iOS Safari）を主要動作環境**として最適化する
- PCブラウザでも全機能が問題なく使えること（レスポンシブ対応）
- タップターゲット最低44px（スマホ片手操作・グローブ着用時を考慮）
- トレーニング中の片手操作を考慮したUI（重要ボタンは画面下部に配置）
- 数値入力フィールドでは数字キーボードを表示する（`inputMode="numeric"`）
- iOS Safariのフォームズーム防止のため入力フォントサイズは16px以上を維持する

### セキュリティ
- 認証済みユーザーのみアクセス可能
- トレーナーは担当顧客のデータのみ操作可能（RLSによる制御）
- 個人情報の最小化（氏名・ニックネームのみ保持）

### プライバシー
- 氏名・ニックネーム以外の個人情報（住所・電話・メール・カード情報）は保存しない
- 顧客のメールアドレスも保存しない（現フェーズでは顧客ログイン機能なし）

### 可用性・コスト
- Vercel Free Tier・Supabase Free Tierの範囲内で運用可能な設計
- 画像・動画ファイルは保存しない（ストレージ節約）

## 7. 制約条件

| 制約 | 詳細 |
|------|------|
| インフラ | Vercel + Supabase（無料枠） |
| ORM | Prisma不使用。@supabase/supabase-jsのみ |
| 個人情報 | 住所・電話・カード情報・顧客メールアドレスは保存禁止 |
| ファイル | 画像・動画のアップロード機能はMVPには含めない |

## 8. 用語定義

| 用語 | 定義 |
|------|------|
| セッション | 1日のトレーニング単位。顧客1人につき1日1セッション |
| 種目 | ベンチプレス・スクワットなどのエクササイズ名 |
| セット | 1種目内の「何kg × 何回」の1単位 |
| トレーナーメモ | セッション全体またはセット単位に付けるフリーテキストのコメント |

## 9. 利用シナリオ（時系列）

### フェーズ1：初期セットアップ（初回のみ）
1. 管理者がSupabaseダッシュボードのAuthentication > Usersからトレーナーアカウントを手動作成
2. トレーナーがアプリにログイン
3. トレーナーが種目マスタを登録（ベンチプレス・スクワットなど）
4. トレーナーが顧客情報を登録

### フェーズ2：日常運用（トレーニング当日）
5. トレーナーがアプリにログイン
6. 顧客一覧から対象顧客を選択
7. 顧客の過去セッション履歴を確認（前回の重量・種目を参考にする）
8. 新規セッションを作成
9. 顧客と相談しながら種目を選択・追加
10. セットごとに重量・回数・メモをその場で記録
11. セッション完了ボタンを押して記録を確定
