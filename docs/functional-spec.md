## 2. 画面仕様

### SCR-01: ログイン

**概要**: トレーナーがメール/パスワードでログインする

**UI要素**
- メールアドレス入力フィールド
- パスワード入力フィールド
- ログインボタン
- エラーメッセージ表示エリア

**挙動**
- ログイン成功 → `/dashboard` へリダイレクト
- ログイン失敗 → エラーメッセージ表示（"メールアドレスまたはパスワードが正しくありません"）
- 認証済みユーザーがアクセス → `/dashboard` へリダイレクト

---

### SCR-02: ダッシュボード

**概要**: ログイン後のトップ画面。直近のセッション・クイックアクションを表示

**UI要素**
- 直近セッション一覧（過去5件）
- 「新規セッション開始」ボタン
- 「顧客一覧」へのリンク

**表示データ**
- セッション日付
- 顧客名（ニックネーム優先）
- 担当トレーナー名

---

### SCR-03: 顧客一覧

**概要**: 登録済み顧客の一覧表示と管理

**UI要素**
- 顧客カード一覧（名前・ニックネーム・最終セッション日）
- 「新規顧客登録」ボタン
- 顧客名での検索フィルタ

**挙動**
- 顧客カードタップ → SCR-05（顧客詳細）へ遷移

---

### SCR-04: 顧客登録

**概要**: 新規顧客の登録

**入力フォーム**
| フィールド | 種別 | 必須 | 備考 |
|-----------|------|------|------|
| 氏名 | テキスト | ○ | 最大50文字 |
| ニックネーム | テキスト | - | 最大30文字。アプリ内での表示名 |
| メモ | テキスト（複数行） | - | トレーナー用の備考。最大500文字 |

**バリデーション**
- 氏名: 必須、1〜50文字
- ニックネーム: 0〜30文字

**挙動**
- 登録成功 → SCR-05（顧客詳細）へリダイレクト
- キャンセル → SCR-03（顧客一覧）へ戻る

---

### SCR-05: 顧客詳細・履歴

**概要**: 顧客プロフィールとセッション履歴の表示

**表示内容**
- 顧客情報（氏名・ニックネーム・メモ）
- 「セッション開始」ボタン
- 「顧客情報編集」ボタン
- セッション履歴一覧（日付・担当トレーナー・種目数）

**挙動**
- セッション行タップ → SCR-08（セッション詳細）へ遷移
- 「セッション開始」タップ → SCR-07（セッション作成）へ、顧客IDを引き継ぎ

---

### SCR-06: 顧客編集

**概要**: 顧客情報の編集・削除

**入力フォーム**: SCR-04と同様

**追加UI**
- 「削除」ボタン（確認ダイアログあり）

**削除時の挙動**
- 確認ダイアログ: "この顧客を削除しますか？関連するセッション記録もすべて削除されます"
- 削除成功 → SCR-03（顧客一覧）へリダイレクト

---

### SCR-07: セッション作成

**概要**: トレーニングセッションの作成（新規記録の起点）

**入力フォーム**
| フィールド | 種別 | 必須 | 備考 |
|-----------|------|------|------|
| 顧客 | セレクト | ○ | 顧客一覧から選択。URL引き継ぎ時は自動選択 |
| 日付 | 日付ピッカー | ○ | デフォルト: 今日 |
| セッションメモ | テキスト（複数行） | - | セッション全体のメモ |

**挙動**
- 作成成功 → SCR-08（セッション詳細・記録）へ遷移

---

### SCR-08: セッション詳細・記録

**概要**: トレーニングセッションのメイン画面。種目とセットを記録する

**表示エリア**
- セッション情報（日付・顧客名・担当トレーナー・セッションメモ）
- 種目リスト（追加済み種目とセット記録）
- 「種目を追加」ボタン

**種目追加フロー**
1. 「種目を追加」タップ → モーダル表示
2. 種目マスタから選択 または フリーテキスト入力
3. 追加 → セット入力エリアが展開

**セット記録UI**（各種目の下に展開）
| フィールド | 種別 | 必須 | 備考 |
|-----------|------|------|------|
| 重量 (kg) | 数値 | - | 小数点1桁まで（0.5kg単位） |
| 回数 | 数値 | - | 整数 |
| メモ | テキスト | - | フォームの注意点など |

- 「セット追加」ボタンで行を追加
- 各セットの削除ボタン
- **保存方式: onBlur保存**
  - 各入力フィールド（重量・回数・メモ）からフォーカスが外れたタイミングで Supabase に UPSERT する
  - 保存中は小さなローディングインジケーターを表示する
  - 保存完了後に「保存済み ✓」を2秒表示して消す

**セッション完了**
- 「セッション完了」ボタン → 完了ステータスに更新、SCR-05（顧客詳細）へ戻る

---

### SCR-09: 種目マスタ管理

**概要**: よく使う種目のマスタデータ管理

**表示内容**
- 種目一覧（種目名・カテゴリ・メモ）
- 「種目追加」ボタン

**種目追加フォーム**（インライン or モーダル）
| フィールド | 種別 | 必須 | 備考 |
|-----------|------|------|------|
| 種目名 | テキスト | ○ | 例: "ベンチプレス" |
| カテゴリ | セレクト | - | 胸・背中・肩・腕・脚・体幹・有酸素 |
| メモ | テキスト | - | 種目の説明や注意点 |

## 3. 認証・認可仕様

### 認証方式
- Supabase Auth（メール/パスワード）
- JWTトークンはSupabaseが管理

### ルート保護
- `(dashboard)` グループ配下のルートはすべて認証必須
- 未認証アクセス → `/login` へリダイレクト（`app/(dashboard)/layout.tsx` のサーバーサイド認証チェックで実装）
- 備考: Vercel Edge Runtimeとの互換性問題により、Next.js Middlewareではなくレイアウトで認証ガードを行う

### ロール設計
```
Role: trainer（現在の実装対象）
  - 全顧客データの読み書き
  - 全セッションデータの読み書き
  - 種目マスタの読み書き

Role: customer（将来拡張）
  - 自分のセッションデータの読み取りのみ
  - 他顧客データへのアクセス不可
```

### RLSポリシー概要
- `profiles`: 自分のプロフィールのみ読み書き可能
- `customers`: `trainer`ロールのみ全操作可能
- `training_sessions`: `trainer`は全操作可、`customer`は自分のセッションのみ読み取り可
- `training_session_exercises`: 親セッションのアクセス権に準ずる
- `training_sets`: 親セッションのアクセス権に準ずる
- `exercises`: 全認証ユーザーが読み取り可能、`trainer`のみ書き込み可能

## 4. データフロー

### セッション記録フロー
```
1. SCR-07でセッション作成 → training_sessionsにレコード挿入
2. SCR-08で種目追加 → training_session_exercisesにレコード挿入
3. セット入力 → training_setsにレコード挿入（即時保存）
4. 「セッション完了」 → training_sessions.status を 'completed' に更新
```

### 自動設定値
- `training_sessions.trainer_id` → ログイン中のユーザーID（`auth.uid()`）
- 作成日時・更新日時 → PostgreSQL `DEFAULT NOW()` / トリガー

## 5. 端末別レイアウト方針

### 画面ごとの主要端末と最適化ポイント

| 画面 | 主要端末 | 最適化ポイント |
|------|---------|--------------|
| SCR-01 ログイン | 両対応 | フォームを中央寄せ、入力サイズ確保 |
| SCR-02 ダッシュボード | iPhone優先 | クイックアクションボタンを大きく・下部に配置 |
| SCR-03 顧客一覧 | 両対応 | スマホ: カードリスト / PC: テーブル形式 |
| SCR-04 顧客登録 | PC優先 | テキスト量が多いため横幅を活かしたフォームレイアウト |
| SCR-05 顧客詳細・履歴 | iPhone優先 | セッション履歴をスクロールしやすいリスト形式 |
| SCR-06 顧客編集 | PC優先 | SCR-04と同様 |
| SCR-07 セッション作成 | iPhone優先 | 顧客選択・日付を大きなタップターゲットで表示 |
| SCR-08 セッション詳細・記録 | **iPhone専用設計** | 重量・回数の数値入力にテンキー表示、セットボタンは親指で届く位置 |
| SCR-09 種目マスタ管理 | PC優先 | 種目の一括管理はPC操作を想定 |

### iOS Safari 固有の対応事項

- **フォームズーム防止**: すべての `<input>` / `<textarea>` のフォントサイズを `16px` 以上に設定
- **セーフエリア対応**: `padding-bottom: env(safe-area-inset-bottom)` をボトムナビゲーション・固定フッターに適用
- **数値入力**: 重量・回数フィールドは `inputMode="numeric"` でテンキーを表示
- **スクロール慣性**: `-webkit-overflow-scrolling: touch` の挙動を考慮したスクロール設計
- **ダブルタップズーム防止**: `touch-action: manipulation` をボタン要素に適用

### レスポンシブブレークポイント方針

```
< 640px  (sm): スマホ縦 — 1カラム、ボトムナビ、タップ最適化
640px〜 (sm+): タブレット以上 — サイドバーナビに切り替え可
1024px〜 (lg): PC — テーブル表示・広いフォームレイアウト
```

## 6. エラーハンドリング方針

| ケース | 対応 |
|--------|------|
| ネットワークエラー | トースト通知 + リトライ促す |
| 認証切れ | `/login` へリダイレクト |
| 権限エラー（403） | "この操作は許可されていません"メッセージ表示 |
| バリデーションエラー | フォームフィールド直下にインラインエラー表示 |
| 予期しないエラー | 汎用エラーメッセージ + エラー詳細はコンソールへ |
